# Codebase Singularity Plan

## Phase 1 Findings
- Stack: Next.js 13 (pages + standalone), Express server, Drizzle ORM, React 18, Wouter client, Zustand, Tailwind.
- Structure: `server/` core logic + services, `shared/` types/schema, `client/src/` UI, `pages/` Next pages API proxy.
- TypeScript strict mode enabled, but `any`/`as any` used in core server.
- Tooling: ESLint Next core vitals; `tsc` requires local bin (script exists but Windows shell lacks `tsc` in PATH). 

## Dependency Graph (Condensed)
- `shared/` schema/types -> `server/` db/services -> `server/routes.ts` -> `pages/api/[...path].ts` -> `client/src/lib/api.ts` -> `client/src/*` UI.

## Rot Centers (High Complexity / Low Maintainability)
- `server/services/statsAggregator.ts` (many `any`, manual data normalization)
- `server/index.ts` (error handling/logging; server lifecycle)
- `server/db.ts` (nullable db, `any` escape hatch)
- `pages/api/[...path].ts` (unsafe casts)
- `client/src/lib/api.ts` (API hooks + data models, likely type drift)

## Source of Truth for Types
- Primary: `shared/schema.ts` + `shared/models/*`.
- Secondary: explicit DTOs in `server/services/*` and `client/src/lib/*` to be aligned to `shared`.

---

## [CRITICAL]
1. Eliminate `any` and `as any` in `server/index.ts`, `server/db.ts`, `pages/api/[...path].ts`.
2. Ensure error handler does not throw after response; add structured logger usage.
3. Make `db` non-nullable or provide typed guard; remove nullable export to avoid runtime crashes.
4. Normalize API contracts between `server/routes.ts` and `client/src/lib/api.ts`.
5. Strictly type `statsAggregator` normalization path and cache.

## [IMPORTANT]
1. Replace console logs with `server/infrastructure/logger.ts`.
2. Add JSDoc for complex normalization/aggregation in `server/services/*`.
3. Align hooks typing in `client/src/lib/api.ts` and `client/src/hooks/use-auth.ts`.
4. Remove dead code: unused imports, redundant helpers.

## [COSMETIC]
1. Consistent formatting and naming.
2. Replace inline `Record<string, any>` with typed interfaces.
3. Remove redundant comments and noise.

---

## Execution Order
1. Foundations: `shared/` types + `server/db.ts` + logger contracts.
2. Core logic: `server/services/*`, `server/routes.ts`.
3. UI: `client/src/lib/*` then `client/src/pages/*` and `client/src/components/*`.
4. Cleanup: remove dead code/logs and normalize formatting.

## Verification

- Fix until clean (exit code 0).
